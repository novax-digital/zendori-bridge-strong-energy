# Demo-Anleitung & Testplan (Checkpoint 1)

End-to-End-Test der Phase 1: Formular- und E-Mail-Ingest → KI-Extraktion → HubSpot-Ticket, plus Paste-Inbox, Dashboard und Auto-Reply. Voraussetzung ist das einmalige Setup (A). Danach ist jeder Testfall in wenigen Minuten durchführbar.

## A. Einmaliges Setup (~30–45 Min)

1. **Supabase-Projekt** anlegen (Region **eu-central-1 Frankfurt**, Plan des Kunden klären).
   - SQL-Editor: erst `supabase/migrations/0001_initial_schema.sql`, dann `0002_phase1.sql` ausführen (beide liegen zur Review vor — **Freigabe = Ausführung**).
   - Authentication → Sign In / Up: **„Allow new users to sign up" deaktivieren**.
   - Authentication → Users → **„Invite user"**: eigene E-Mail einladen, Passwort setzen.
   - API-Keys notieren: URL, Publishable/Anon-Key, Secret/Service-Role-Key.
2. **Vercel-Projekt** (Pro): Repo importieren, **Root Directory `apps/web`**, alle Variablen aus `.env.example` setzen (inkl. `CRON_SECRET` generieren), Domain `strongenergy.zendori.ai` per CNAME verbinden. Deploy. Details: README → Deployment.
3. **Smoke:** `https://strongenergy.zendori.ai/healthz` → `{"status":"ok"}`. Login mit dem eingeladenen Nutzer → leerer Posteingang.
4. **HubSpot verbinden:** Private App im Kunden-Portal mit Scopes **`tickets`** + **`crm.objects.contacts.read`** + **`crm.objects.contacts.write`**; Token als `HUBSPOT_TOKEN` in Vercel setzen (Redeploy). Dann Dashboard → Einstellungen:
   - „Verbindung testen / Pipelines laden" → Pipeline + Eingangs-Stage wählen, speichern.
   - „Custom Properties anlegen" → legt `zendori_ref` (unique) + `zendori_source` an.
5. **Postfach anlegen:** Einstellungen → Postfächer → Formular ausfüllen (IMAP/SMTP-Host, Ports, Adresse, Passwort), speichern → **„Verbindung testen"** muss für IMAP und SMTP grün sein. Auto-Reply nach Wunsch aktivieren.
6. **Formular-Key erzeugen:** Einstellungen → Formular-API-Keys → Label + erlaubte Domain(s) → Key **sofort kopieren** (wird nur einmal angezeigt).

## B. Testfälle

| --- | -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | **Formular → Ticket**            | `curl -X POST https://strongenergy.zendori.ai/api/ingest/form -H 'content-type: application/json' -H 'x-zendori-key: <KEY>' -d '{"name":"Max Test","email":"max@example.com","nachricht":"Meine Wallbox EnergyBox 22 zeigt seit heute früh einen roten Blinkcode und lädt nicht. Bitte um Rückruf.","request_id":"test-001"}'` | Antwort 202 mit `correlation_id`. Posteingang: Eintrag „Formular", Status läuft binnen ~1 Min auf **Ticket erstellt**. HubSpot: neues Ticket `ZV1-0001` in der gewählten Pipeline, Kontakt `max@example.com` verknüpft, `zendori_source=form`. |
| 2   | **Idempotenz**                   | Denselben curl (gleiche `request_id`) erneut senden                                                                                                                                                                                                                                                                            | 202 `bereits_verarbeitet`; **kein** zweites Ticket.                                                                                                                                                                                            |
| 3   | **E-Mail → Ticket + Auto-Reply** | Von einer privaten Adresse eine Mail mit konkretem Anliegen ans Testpostfach schicken                                                                                                                                                                                                                                          | Binnen ~1–2 Min im Posteingang (Kanal E-Mail), dann Ticket in HubSpot; Absender erhält Auto-Reply mit `[ZV1-…]` im Betreff (sofern aktiviert).                                                                                                 |

| 6   | **Spam**                         | Formular-POST mit Werbetext (SEO/Backlinks)                                                                                                                                                                                                                                                                                    | Status **Spam**, kein Ticket.                                                                                                                                                                                                                  |
| 7   | **Paste-Inbox**                  | Dashboard → „Nachricht einfügen": Telefonnotiz einfügen (mit Rückrufnummer), Analysieren → Vorschau prüfen/ergänzen → „Ticket erstellen"                                                                                                                                                                                       | Vorschau zeigt extrahierte Felder (fehlende amber markiert); nach Erstellen Ticket in HubSpot mit `zendori_source=paste`.                                                                                                                      |
| 8   | **Erneut verarbeiten**           | In einer Detailansicht „Erneut verarbeiten" klicken                                                                                                                                                                                                                                                                            | Status springt auf Eingegangen und durchläuft die Pipeline erneut; kein doppeltes HubSpot-Ticket (gleiche `zendori_ref`).                                                                                                                      |

| 10  | **Fehlerpfad sichtbar**          | `HUBSPOT_TOKEN` in Vercel kurz entfernen (Redeploy), Testfall 1 wiederholen, Token wieder setzen                                                                                                                                                                                                                               | Nachricht endet nach 5 Versuchen als **Fehlgeschlagen** mit Fehlertext in der Detailansicht — kein stiller Verlust. Danach „Erneut verarbeiten" → Ticket entsteht.                                                                             |
| 11  | **Website-Formulare**            | Die beiden Bestandsformulare per Snippet aus `docs/formular-einbindung.md` anbinden (Domain vorher beim Key eintragen)                                                                                                                                                                                                         | Beide Formulare liefern Tickets; Honeypot-Feld `website` bleibt unsichtbar/leer.                                                                                                                                                               |

## C. Bekannte Phase-1-Grenzen (kommen in 1.5/2)

Dedup-Engine (aktuell Pass-through, jede Nachricht = neues Ticket), „Braucht Info"-Queue mit Rückfragen-Versand, Duplikat-Review/Merge, Lösch-Jobs, signierte Anhang-Downloads im Dashboard, Telefon & WhatsApp.
